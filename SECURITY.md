# Security Policy

## Supported Versions

The current `main` branch and the latest published npm release are the supported security baseline.

## Reporting A Vulnerability

- Do not open public GitHub issues for sensitive vulnerabilities.
- Report the issue privately to the maintainers through the repository security advisory flow or a private communication channel.
- Include platform, Node version, reproduction steps and whether the issue affects screen capture, synthetic input, or package/runtime trust boundaries.

## Scope

Security-sensitive areas include:

- Desktop input execution.
- Screen capture and resource exposure.
- Trust boundaries around MCP client configuration.
- Runtime resolution and package publishing.
